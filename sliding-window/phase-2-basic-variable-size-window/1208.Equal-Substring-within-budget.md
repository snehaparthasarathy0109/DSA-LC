# 1208. Get Equal Substrings Within Budget

## Problem
You are given two strings `s` and `t` of the same length, and an integer `maxCost`.

Changing `s[i]` to `t[i]` costs:

```python
abs(ord(s[i]) - ord(t[i]))
````

Return the maximum length of a substring of `s` that can be changed to the corresponding substring of `t` with total cost at most `maxCost`.

---

# Key Idea

For each position `i`, define:

```python
cost[i] = abs(ord(s[i]) - ord(t[i]))
```

Now the problem becomes:

> Find the longest contiguous subarray of `cost` whose sum is `<= maxCost`.

That is a classic sliding window pattern.

---

# Sliding Window Strategy

We maintain a window `[left, right]` such that:

```python
currentCost <= maxCost
```

As we move `right`:

* add the new conversion cost

If the total exceeds `maxCost`:

* shrink from the left until the window is valid again

At every valid step:

* update the maximum length

---

# Solution

```python
class Solution:
    def equalSubstring(self, s: str, t: str, maxCost: int) -> int:
        left = 0
        currentCost = 0
        maxLen = 0

        for right in range(len(s)):
            currentCost += abs(ord(s[right]) - ord(t[right]))

            while currentCost > maxCost:
                currentCost -= abs(ord(s[left]) - ord(t[left]))
                left += 1

            maxLen = max(maxLen, right - left + 1)

        return maxLen
```

---

# Why This Works

## Step 1: Add the cost for the new character

When we expand the window to the right:

```python
currentCost += abs(ord(s[right]) - ord(t[right]))
```

---

## Step 2: Shrink if over budget

If the total cost becomes too large:

```python
while currentCost > maxCost:
```

we move `left` forward and remove the leftmost cost from the window.

---

## Step 3: Update the answer

Once the window is valid again, its length is:

```python
right - left + 1
```

So we update:

```python
maxLen = max(maxLen, right - left + 1)
```

# Why This Is Sliding Window and Not DP

This problem asks for:

* a **substring** → contiguous
* longest window such that
* total cost stays within a limit

This is a classic “longest valid window” problem.

Use sliding window when:

* the problem is about a contiguous window
* you can grow/shrink the window
* the condition depends on a running sum or count

Use DP when:

* the answer depends on branching subproblems
* there is no simple moving-window structure
