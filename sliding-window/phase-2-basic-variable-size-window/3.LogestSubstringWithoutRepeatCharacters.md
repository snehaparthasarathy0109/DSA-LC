# 3. Longest Substring Without Repeating Characters

## Problem
Given a string `s`, find the length of the **longest substring** without repeating characters.

A **substring** is contiguous, so the characters must be next to each other.

---

## Example
```python
Input: s = "abcabcbb"
Output: 3
````

Explanation:

The longest substring without repeating characters is `"abc"`, so the answer is `3`.

---

# Key Idea

We want the **longest contiguous window** where all characters are unique.

This is a classic **variable-size sliding window** problem.

We keep:

* `left` = start of the current window
* `right` = end of the current window
* `dictionary` = frequency of characters in the current window
* `maxLen` = longest valid window seen so far

As we move `right` forward:

* add the new character to the window
* if it creates a duplicate, shrink the window from the left until the window becomes valid again
* update `maxLen`

---

# Correct Code

```python
from collections import defaultdict

class Solution:
    def lengthOfLongestSubstring(self, s: str) -> int:
        dictionary = defaultdict(int)
        left = 0
        maxLen = 0

        for right in range(len(s)):
            dictionary[s[right]] += 1

            while dictionary[s[right]] > 1:
                dictionary[s[left]] -= 1
                if dictionary[s[left]] == 0:
                    del dictionary[s[left]]
                left += 1

            maxLen = max(maxLen, right - left + 1)

        return maxLen
```

---

# Why This Works

## 1. Expand the window

For every `right`, we include `s[right]` in the window:

```python
dictionary[s[right]] += 1
```

---

## 2. If duplicate appears, shrink the window

If the count of `s[right]` becomes greater than 1, the window now has a repeated character.

So we move `left` forward and remove characters until the duplicate is gone:

```python
while dictionary[s[right]] > 1:
    dictionary[s[left]] -= 1
    if dictionary[s[left]] == 0:
        del dictionary[s[left]]
    left += 1
```

This guarantees that after the `while` loop, the window has all unique characters again.

---

## 3. Update the answer

Once the window is valid, its length is:

```python
right - left + 1
```

So we update:

```python
maxLen = max(maxLen, right - left + 1)
```

---

# Why We Use `while` and Not `if`

A duplicate may require removing **more than one** character.

Example:

```python
s = "abba"
```

When we reach the second `'b'`, just removing one character is not always enough. We need to keep shrinking until the window becomes valid again.

That is why we use:

```python
while dictionary[s[right]] > 1:
```

not just:

```python
if dictionary[s[right]] > 1:
```

---

# Intuition in One Sentence

We keep expanding the window to the right, and whenever a duplicate appears, we shrink from the left until the window has unique characters again.

---

# Time Complexity

```python
O(n)
```

Why?

* each character is added once by `right`
* each character is removed at most once by `left`

So total work is linear.

---

# Space Complexity

```python
O(min(n, charset))
```

In Python with general strings, this means the dictionary stores at most the number of distinct characters in the current window.

---

# Common Mistakes

## 1. Using `if` instead of `while`

Wrong:

```python
if dictionary[s[right]] > 1:
```

Correct:

```python
while dictionary[s[right]] > 1:
```

Because sometimes one removal is not enough.

---

## 2. Returning `len(dictionary)`

Wrong because that only gives the distinct count in the **final** window, not the longest valid window seen overall.

You must track:

```python
maxLen
```

---

## 3. Using the index as the dictionary key

Wrong:

```python
dictionary[right] += 1
```

Correct:

```python
dictionary[s[right]] += 1
```

Because we care about character frequencies, not index frequencies.

# Short interview explanation

This is a variable-size sliding window problem.
I keep a window with unique characters using a frequency map.
As I expand the window with `right`, if a duplicate appears, I shrink from the left until the window is valid again.
At every valid step, I update the maximum window length.
