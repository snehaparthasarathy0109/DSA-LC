This is a **variable-size sliding window** problem.

## Why?

You are walking right through a contiguous block of trees, and you can keep going as long as the window contains **at most 2 fruit types**.

So the problem becomes:

> Find the longest contiguous subarray with **at most 2 distinct numbers**.

---

## Solution

```python
from collections import defaultdict

class Solution:
    def totalFruit(self, fruits: List[int]) -> int:
        count = defaultdict(int)
        left = 0
        maxLen = 0

        for right in range(len(fruits)):
            count[fruits[right]] += 1

            while len(count) > 2:
                count[fruits[left]] -= 1
                if count[fruits[left]] == 0:
                    del count[fruits[left]]
                left += 1

            maxLen = max(maxLen, right - left + 1)

        return maxLen
```

---

## How it works

We keep a window from `left` to `right`.

* Add the fruit at `right`
* If the window now has more than 2 fruit types, shrink from the left
* Once valid again, update the maximum length

The dictionary stores how many of each fruit type are in the current window.

## Dry run

### Example

```python
fruits = [1,2,3,2,2]
```

Start:

* `left = 0`
* `count = {}`
* `maxLen = 0`

### `right = 0`, fruit = 1

Window: `[1]`
`count = {1: 1}`
valid, at most 2 types

`maxLen = 1`

### `right = 1`, fruit = 2

Window: `[1,2]`
`count = {1: 1, 2: 1}`
valid

`maxLen = 2`

### `right = 2`, fruit = 3

Window: `[1,2,3]`
`count = {1: 1, 2: 1, 3: 1}`
now 3 types, invalid

Shrink from left:

* remove `1`
* `count = {2: 1, 3: 1}`
* `left = 1`

Now window is `[2,3]`, valid again

`maxLen = 2`

### `right = 3`, fruit = 2

Window: `[2,3,2]`
`count = {2: 2, 3: 1}`
valid

`maxLen = 3`

### `right = 4`, fruit = 2

Window: `[2,3,2,2]`
`count = {2: 3, 3: 1}`
valid

`maxLen = 4`

Answer = `4`

## Why this is sliding window

Because:

* you must pick from a contiguous run of trees
* you move only to the right
* the condition is “at most 2 distinct fruit types”

That is exactly the pattern:

> longest subarray with at most K distinct elements

Here, `K = 2`.

---

## Time complexity

* `O(n)`

Each fruit enters the window once and leaves at most once.

## Space complexity

* `O(1)` effectively, since the map holds at most 3 keys during processing

---

## Pattern to remember

This problem is the classic:

**Longest subarray with at most K distinct elements**

Template:

```python
for right in range(len(arr)):
    add arr[right]

    while window invalid:
        remove arr[left]
        left += 1

    update answer
```

Because `right - left + 1` is the **number of trees in the current valid window**.

In this problem, the window represents the contiguous run of trees you can pick from.

If the current window is from index `left` to index `right`, then the trees are:

* `left`
* `left + 1`
* `left + 2`
* ...
* `right`

The count of indices from `left` to `right` inclusive is:

```python
right - left + 1
```

---

## Why `+ 1`?

Because both ends are included.

Example:

```python
left = 2
right = 5
```

The trees are at indices:

```python
2, 3, 4, 5
```

That is `4` trees.

And:

```python
5 - 2 + 1 = 4
```

If you did only:

```python
right - left
```

you would get:

```python
5 - 2 = 3
```

which misses one tree.

---

## Why does this equal the amount of fruit?

Because the problem says:

> you must pick exactly **one fruit from every tree** while moving to the right

So if your valid window has length 4, that means you can collect fruit from 4 trees, which means 4 fruits total.

One tree gives one fruit.

So:

```python
number of fruits collected = number of trees in the valid window = right - left + 1
```

## Example

```python
fruits = [1,2,3,2,2]
```

Suppose the valid window is:

```python
[2,3,2,2]
```

This window starts at index `1` and ends at index `4`.

So:

```python
left = 1
right = 4
```

Length is:

```python
4 - 1 + 1 = 4
```

That means you can collect 4 fruits.

Why? Because you pick one fruit from each of these 4 trees:

* tree 1
* tree 2
* tree 3
* tree 4

---

## Intuition

The sliding window is literally the stretch of trees you are currently able to walk through without breaking the “at most 2 fruit types” rule.

So the best answer is just the **largest valid window length**.

And window length is:

```python
right - left + 1
```
