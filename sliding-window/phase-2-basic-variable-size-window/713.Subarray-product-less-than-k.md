This is a **variable-size sliding window** problem.

Because all numbers are positive, if the product gets too large, moving `left` rightward will only make the product smaller. That monotonic behavior is what makes sliding window work.

# 713. Subarray Product Less Than K

## Problem
Given an array of positive integers `nums` and an integer `k`, return the number of contiguous subarrays where the product of all the elements is **strictly less than** `k`.

---

## Example
```python
nums = [10, 5, 2, 6]
k = 100
````

Output:

```python
8
```

Valid subarrays:

```python
[10], [5], [2], [6], [10,5], [5,2], [2,6], [5,2,6]
```

---

# Key Idea

We maintain a sliding window `[left, right]` whose product is always `< k`.

As we expand `right`:

* multiply in `nums[right]`

If product becomes `>= k`:

* shrink from the left until product is `< k` again

Once the window is valid, the number of valid subarrays ending at `right` is:

```python
right - left + 1
```

---

# Why `right - left + 1`?

Suppose after shrinking, the valid window is:

```python
nums[left:right+1]
```

Then every subarray ending at `right` and starting anywhere from `left` to `right` is valid:

* `[nums[right]]`
* `[nums[right-1], nums[right]]`
* ...
* `[nums[left], ..., nums[right]]`

That is exactly:

```python
right - left + 1
```

subarrays.

---

# Solution

```python id="d0r8mn"
class Solution:
    def numSubarrayProductLessThanK(self, nums: List[int], k: int) -> int:
        if k <= 1:
            return 0

        product = 1
        left = 0
        count = 0

        for right in range(len(nums)):
            product *= nums[right]

            while product >= k:
                product //= nums[left]
                left += 1

            count += right - left + 1

        return count
```

---

# Why `k <= 1` returns 0

All numbers in `nums` are positive integers, so every subarray product is at least `1`.

So if:

```python
k <= 1
```

there is no way for any product to be strictly less than `k`.

Example:

* smallest possible product is `1`
* but `1 < 1` is false

So answer is `0`.

# Intuition Behind the Counting

At each `right`, once the window is valid, every shorter suffix of that window is also valid because all numbers are positive.

So instead of listing all subarrays one by one, we can count them in one shot.

That is the main trick.

---

# Why Sliding Window Works Here

Sliding window works because all values are positive.

If product is too large:

* removing numbers from the left makes it smaller

That monotonic behavior is necessary.

If negatives or zeroes were involved in a more complex way, this pattern would break.

---

# Time Complexity

```python
O(n)
```

Why?

* each element is multiplied in once
* each element is divided out at most once

So both pointers move across the array only once.

---

# Space Complexity

```python
O(1)
```

Only a few variables are used.

# Short interview explanation

This is a variable-size sliding window problem.
I maintain a window whose product is less than `k`.
For each `right`, I multiply in `nums[right]`, and if the product becomes too large, I shrink from the left until it is valid again.
Once valid, the number of subarrays ending at `right` is `right - left + 1`, so I add that to the answer.
