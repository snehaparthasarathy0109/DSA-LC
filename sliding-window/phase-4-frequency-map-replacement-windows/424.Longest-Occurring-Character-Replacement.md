# 424. Longest Repeating Character Replacement

## Problem
You are given a string `s` and an integer `k`.

You can change at most `k` characters in the string. Each change can turn a character into any other uppercase English character.

Return the length of the **longest substring** that can be made of the **same repeating character** after performing at most `k` replacements.

---

## Example 1

```python
s = "ABAB"
k = 2
````

Output:

```python
4
```

Explanation:

We can replace the two `'A'` characters with `'B'`, or the two `'B'` characters with `'A'`, so the whole substring becomes the same character.

---

## Example 2

```python
s = "AABABBA"
k = 1
```

Output:

```python
4
```

Explanation:

One optimal answer is length `4`.

---

# Key Idea

This is a **variable-size sliding window** problem.

We want the **longest window** such that we can make every character in that window the same using at most `k` replacements.

For a window:

```python
s[left:right+1]
```

let:

* `window_len = right - left + 1`
* `max_freq = frequency of the most common character in the window`

Then the number of replacements needed is:

```python
window_len - max_freq
```

Why?

Because the cheapest strategy is:

* keep the character that appears the most
* replace all other characters

So the window is valid if:

```python
window_len - max_freq <= k
```

If it becomes invalid, we shrink it from the left.

---

# Solution Code

```python
from collections import defaultdict

class Solution:
    def characterReplacement(self, s: str, k: int) -> int:
        count = defaultdict(int)
        left = 0
        max_freq = 0
        max_len = 0

        for right in range(len(s)):
            count[s[right]] += 1
            max_freq = max(max_freq, count[s[right]])

            while (right - left + 1) - max_freq > k:
                count[s[left]] -= 1
                left += 1

            max_len = max(max_len, right - left + 1)

        return max_len
```

---

# Why This Works

## Step 1: Expand the window

We move `right` from left to right, adding characters into the current window.

```python
count[s[right]] += 1
```

This tracks how many times each character appears in the window.

---

## Step 2: Track the most frequent character

We keep track of the highest frequency of any character we have seen in the current window:

```python
max_freq = max(max_freq, count[s[right]])
```

This tells us the size of the character group we can keep unchanged.

---

## Step 3: Check how many replacements are needed

The current window has length:

```python
right - left + 1
```

If the most frequent character appears `max_freq` times, then all the other characters must be replaced.

So:

```python
replacements_needed = (right - left + 1) - max_freq
```

If that number is greater than `k`, the window is invalid.

That is why we use:

```python
while (right - left + 1) - max_freq > k:
```

This means:

> while this window needs more than `k` replacements, shrink it

---

## Step 4: Shrink the window when invalid

If the window needs too many replacements, move `left` forward:

```python
count[s[left]] -= 1
left += 1
```

This reduces the window size until it becomes valid again.

---

## Step 5: Update the answer

Whenever the window is valid, update the longest valid length:

```python
max_len = max(max_len, right - left + 1)
```

---

# Understanding This Line

```python
while (right - left + 1) - max_freq > k:
```

This is the most important line in the solution.

Let’s break it down.

## `right - left + 1`

This is the length of the current window.

If:

```python
left = 2
right = 5
```

then the indices are:

```python
2, 3, 4, 5
```

That is 4 characters, so:

```python
5 - 2 + 1 = 4
```

---

## `max_freq`

This is the frequency of the most common character in the window.

Example:

If the window is:

```python
"AABAB"
```

then the counts are:

* `A = 3`
* `B = 2`

So:

```python
max_freq = 3
```

---

## `(window length) - max_freq`

This gives the number of characters that are **not** the most common character.

For `"AABAB"`:

* window length = 5
* max frequency = 3

So:

```python
5 - 3 = 2
```

That means 2 characters must be replaced to make the whole window one repeated character.

---

## Why compare to `k`?

Because the problem allows at most `k` replacements.

So:

* if `window_len - max_freq <= k`, the window is valid
* if `window_len - max_freq > k`, the window is invalid

That is exactly what this line checks:

```python
while (right - left + 1) - max_freq > k:
```

# Why We Do Not Decrease `max_freq`

This is a common question.

We only update `max_freq` like this:

```python
max_freq = max(max_freq, count[s[right]])
```

We do **not** recompute it when shrinking the window.

That means `max_freq` can become a little stale.

That is okay.

Why?

Because even if `max_freq` is slightly larger than the true current max frequency, the algorithm still works correctly. It may allow the window to stay a little larger temporarily, but it never causes us to miss the correct answer.

This avoids recomputing the maximum frequency every time, which keeps the solution efficient.

---

# Time Complexity

## `O(n)`

Why?

* `right` moves from left to right once
* `left` also moves from left to right at most once

So each character is processed at most twice.

---

# Space Complexity

## `O(1)`

The string contains only uppercase English letters, so the hashmap stores at most 26 characters.

That is constant space.

---

# Pattern to Remember

This problem follows the pattern:

> Longest valid substring under a replacement budget

The template is:

```python
for right in range(len(s)):
    add s[right]
    update max frequency

    while window is invalid:
        remove s[left]
        left += 1

    update answer
```

And the validity condition is:

```python
window_size - max_freq <= k
```

---

# Common Mistakes

## 1. Not understanding `window_size - max_freq`

This is the number of characters that need to be replaced.

## 2. Using the wrong shrink condition

The correct condition is:

```python
while (right - left + 1) - max_freq > k:
```

Not `>= k`.

If replacements needed is exactly `k`, the window is still valid.

## 3. Recomputing max frequency every time

That would still work, but it is less efficient.

---

# Short Interview Explanation

This is a variable-size sliding window problem.
For each window, I track character frequencies and the count of the most frequent character.
If the window size minus that max frequency is greater than `k`, then the window needs too many replacements, so I shrink it from the left.
At every valid step, I update the maximum window length.
