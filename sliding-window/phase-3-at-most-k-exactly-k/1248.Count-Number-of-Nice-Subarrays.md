You’re mixing up two different problems.

Your code is trying to use:

* a frequency map of values
* distinct-count style logic

But **`numberOfSubarrays`** is the “nice subarrays” problem, where you need to count subarrays with **exactly `k` odd numbers**.

So the thing to track is **how many odd numbers are in the window**, not frequencies of the actual values.

## What is wrong

### 1. Wrong thing being counted

This part:

```python
freq[nums[right]] += 1 
if freq[nums[right]] // 2 != 0:
    count += 1
```

is not counting odd numbers.

It is counting how many times a value appears, which is unrelated.

To check whether a number is odd, use:

```python
nums[right] % 2 == 1
```

---

### 2. Wrong shrink condition

You wrote:

```python
while count > 3:
```

That should depend on `k`, not `3`.

---

### 3. Wrong pattern for “exactly k”

For exactly `k` odd numbers, the easiest sliding-window pattern is:

[
\text{exactly}(k) = \text{atMost}(k) - \text{atMost}(k-1)
]

So we should write a helper that counts subarrays with **at most `k` odd numbers**.

---

# Correct solution

```python id="lsq0gq"
class Solution:
    def numberOfSubarrays(self, nums: List[int], k: int) -> int:
        def atMost(k):
            left = 0
            count_odds = 0
            res = 0

            for right in range(len(nums)):
                if nums[right] % 2 == 1:
                    count_odds += 1

                while count_odds > k:
                    if nums[left] % 2 == 1:
                        count_odds -= 1
                    left += 1

                res += right - left + 1

            return res

        return atMost(k) - atMost(k - 1)
```

---

# Why this works

## `atMost(k)`

Counts subarrays with **at most `k` odd numbers**.

For each `right`:

* add `nums[right]`
* if it is odd, increase `count_odds`
* if odds become too many, shrink from left
* once valid, all subarrays ending at `right` and starting from `left ... right` are valid

So add:

```python id="h172xy"
right - left + 1
```

---

## Exactly `k`

Then:

```python id="i9f8b3"
exactly_k = atMost(k) - atMost(k - 1)
```

because:

* `atMost(k)` includes subarrays with 0, 1, 2, ..., `k` odd numbers
* `atMost(k - 1)` includes subarrays with 0, 1, 2, ..., `k-1` odd numbers

Subtracting leaves only subarrays with exactly `k` odd numbers.

---

# If you want the smallest fix conceptually

Your code should not use `freq` at all. Replace it with an odd counter.

# Dry run example

For:

```python id="bt1h3m"
nums = [1,1,2,1,1]
k = 3
```

Answer is `2` because the nice subarrays are:

* `[1,1,2,1]`
* `[1,2,1,1]`

Using the helper:

* `atMost(3) = 14`
* `atMost(2) = 12`

So:

```python id="ps1huh"
14 - 12 = 2
```

# In short

Your code is wrong because:

* it tracks frequency of values instead of number of odd numbers
* it uses `3` instead of `k`
* it tries to solve “exactly k” directly with the wrong window condition

Use this instead:
