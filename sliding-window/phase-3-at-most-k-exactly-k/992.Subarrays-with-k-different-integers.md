This is a classic **sliding window counting** problem.

The hard part is:

* counting subarrays with **exactly** `k` distinct integers is awkward directly
* but counting subarrays with **at most** `k` distinct integers is easy

So the trick is:

[
\text{exactly}(k) = \text{atMost}(k) - \text{atMost}(k-1)
]

````md id="4r9x1p"
# 992. Subarrays with K Different Integers

## Problem
Given an integer array `nums` and an integer `k`, return the number of subarrays with **exactly `k` distinct integers**.

A subarray must be contiguous.

---

# Key Insight

Counting subarrays with **exactly `k` distinct integers** directly is difficult.

Instead, count:

- subarrays with **at most `k`** distinct integers
- subarrays with **at most `k - 1`** distinct integers

Then subtract:

```python
exactly_k = atMost(k) - atMost(k - 1)
````

Why?

Because:

* `atMost(k)` includes subarrays with 1, 2, ..., `k` distinct integers
* `atMost(k - 1)` includes subarrays with 1, 2, ..., `k - 1` distinct integers

So the only subarrays left after subtraction are those with **exactly `k`** distinct integers.

---

# Main Formula

```python id="h3xuyr"
exactly(k) = atMost(k) - atMost(k - 1)
```

---

# Step 1: Solve `atMost(k)`

We use a sliding window.

For each `right`, maintain a window with **at most `k` distinct numbers**.

If the window becomes invalid:

* shrink from the left until it becomes valid again

Once valid, the number of valid subarrays ending at `right` is:

```python id="fmx8sv"
right - left + 1
```

Why?

Because every subarray ending at `right` and starting anywhere from `left` to `right` is valid.

So we add:

```python id="n0aqz0"
count += right - left + 1
```

---

# Full Solution

```python id="we3638"
from collections import defaultdict

class Solution:
    def subarraysWithKDistinct(self, nums: List[int], k: int) -> int:
        def atMost(k):
            count = defaultdict(int)
            left = 0
            result = 0

            for right in range(len(nums)):
                count[nums[right]] += 1

                while len(count) > k:
                    count[nums[left]] -= 1
                    if count[nums[left]] == 0:
                        del count[nums[left]]
                    left += 1

                result += right - left + 1

            return result

        return atMost(k) - atMost(k - 1)
```

---

# Why `right - left + 1`?

Suppose the current valid window is:

```python id="i0rke5"
nums[left:right+1]
```

Then all these subarrays end at `right` and are valid:

* `[nums[right]]`
* `[nums[right-1], nums[right]]`
* `[nums[right-2], ..., nums[right]]`
* ...
* `[nums[left], ..., nums[right]]`

That is exactly:

```python id="ktshm3"
right - left + 1
```

subarrays.

# Why This Is Sliding Window

This problem is about counting **contiguous subarrays** under a condition involving the number of distinct elements.

That usually suggests sliding window.

The direct “exactly k” version is hard, but the “at most k” version is a standard variable-size sliding window:

* expand with `right`
* shrink with `left` if invalid
* count valid endings

---

# Time Complexity

Each `atMost(k)` call is:

```python id="6tmk8j"
O(n)
```

because:

* each element enters the window once
* each element leaves the window at most once

We call it twice, so total is still:

```python id="z1mz10"
O(n)
```

---

# Space Complexity

The hashmap stores frequencies of elements in the current window.

Worst case:

```python id="sy9mq4"
O(n)
```

if all elements are different.

If values are bounded, you could describe it as `O(k)` to `O(n)` depending on the context.

# Intuition in One Sentence

Count subarrays with at most `k` distinct integers, count subarrays with at most `k - 1` distinct integers, and subtract to get exactly `k`.

```python id="ntr8b2"
from collections import defaultdict

class Solution:
    def subarraysWithKDistinct(self, nums: List[int], k: int) -> int:
        def atMost(k):
            count = defaultdict(int)
            left = 0
            result = 0

            for right in range(len(nums)):
                count[nums[right]] += 1

                while len(count) > k:
                    count[nums[left]] -= 1
                    if count[nums[left]] == 0:
                        del count[nums[left]]
                    left += 1

                result += right - left + 1

            return result

        return atMost(k) - atMost(k - 1)
```
