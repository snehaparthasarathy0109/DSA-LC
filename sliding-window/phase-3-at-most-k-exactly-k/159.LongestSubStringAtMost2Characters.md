# Longest Substring with At Most Two Distinct Characters

## Problem
Given a string `s`, return the length of the **longest substring** that contains **at most 2 distinct characters**.

A substring must be **contiguous**.

---

## Solution Code

```python
from collections import defaultdict

class Solution:
    def lengthOfLongestSubstringTwoDistinct(self, s: str) -> int:
        left = 0
        maxLen = 0
        dictionary = defaultdict(int) 

        for right in range(len(s)):
            dictionary[s[right]] += 1 

            while len(dictionary) > 2:
                dictionary[s[left]] -= 1
                if dictionary[s[left]] == 0:
                    del dictionary[s[left]] 
                
                left += 1 

            maxLen = max(maxLen, right - left + 1) 
        return maxLen
````

---

# Intuition

We want the **longest contiguous window** that contains no more than **2 distinct characters**.

This is a classic **variable-size sliding window** problem.

We use:

* `left` to mark the start of the window
* `right` to mark the end of the window
* a hashmap `dictionary` to count character frequencies in the current window
* `maxLen` to store the longest valid window seen so far

As we move `right` forward:

* add the new character into the window
* if the window has more than 2 distinct characters, shrink it from the left
* once valid again, update the answer

---

# Why Sliding Window Works

The problem is asking for a **longest substring**, and substring means **contiguous**.

That is the main clue that sliding window may work.

The condition we must maintain is:

```python
at most 2 distinct characters
```

So we:

1. expand the window to the right
2. check if it became invalid
3. shrink from the left until valid again
4. record the largest valid window

---

# What the Hashmap Stores

```python
dictionary[char] = count of char in the current window
```

Example:

If the current window is:

```python
"ece"
```

then:

```python
dictionary = {'e': 2, 'c': 1}
```

This tells us:

* which characters are in the current window
* how many times they appear
* how many distinct characters exist using `len(dictionary)`

---

# Step-by-Step Explanation

## 1. Expand the window

For each `right`, include `s[right]` in the current window:

```python
dictionary[s[right]] += 1
```

---

## 2. Check if window is invalid

If the number of distinct characters becomes greater than 2:

```python
while len(dictionary) > 2:
```

then the window is invalid and must be shrunk.

---

## 3. Shrink from the left

We remove characters from the left side one at a time:

```python
dictionary[s[left]] -= 1
```

If a character count becomes 0, it is no longer in the window, so we remove it from the dictionary:

```python
if dictionary[s[left]] == 0:
    del dictionary[s[left]]
```

Then move `left` forward:

```python
left += 1
```

We keep doing this until the window has at most 2 distinct characters again.

---

## 4. Update the answer

Once the window is valid, compute its length:

```python
right - left + 1
```

and update:

```python
maxLen = max(maxLen, right - left + 1)
```

---

# Why `right - left + 1`?

If the window starts at `left` and ends at `right`, then the indices included are:

```python
left, left + 1, ..., right
```

That is an inclusive range, so the number of characters is:

```python
right - left + 1
```

Example:

```python
left = 2
right = 5
```

Indices are:

```python
2, 3, 4, 5
```

That is 4 characters.

And:

```python
5 - 2 + 1 = 4
```

# Why We Delete Characters with Count 0

Suppose a character is no longer inside the current window.

Then its count becomes 0.

If we leave it in the dictionary, `len(dictionary)` would still count it as a distinct character, which would be wrong.

That is why we do:

```python
if dictionary[s[left]] == 0:
    del dictionary[s[left]]
```

---

# Why `while` and Not `if`?

We use:

```python
while len(dictionary) > 2:
```

because removing one character may not be enough.

Example:

If the window contains 3 distinct characters, and the leftmost character still remains elsewhere in the window, then removing one occurrence does not reduce the distinct count yet.

So we may need to shrink multiple times.

---

# Time Complexity

## `O(n)`

Each character is:

* added once when `right` moves
* removed at most once when `left` moves

Even though there is a nested `while`, both pointers only move forward, so the total work is linear.

---

# Space Complexity

## `O(1)`

The dictionary stores at most 3 characters during processing, and only 2 in a valid window.

So the extra space is constant.

---

# Pattern

This problem is a special case of:

> Longest substring with at most `K` distinct characters

Here, `K = 2`.

General template:

```python
for right in range(len(s)):
    add s[right]

    while window is invalid:
        remove s[left]
        left += 1

    update answer
```

---

# Common Mistakes

## 1. Forgetting to delete keys with count 0

Then `len(dictionary)` becomes incorrect.

## 2. Using `if` instead of `while`

One removal may not be enough to make the window valid.

## 3. Returning the final window length

You need to track the maximum across all valid windows, not just the last one.
