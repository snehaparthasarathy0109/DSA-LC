# Longest Substring with At Most K Distinct Characters

## Problem
Given a string `s` and an integer `k`, return the length of the **longest substring** that contains **at most `k` distinct characters**.

A substring must be **contiguous**.

---

# Code

```python
from collections import defaultdict

class Solution:
    def lengthOfLongestSubstringKDistinct(self, s: str, k: int) -> int:
        dictionary = defaultdict(int)
        left = 0
        maxLen = 0

        for right in range(len(s)):
            dictionary[s[right]] += 1

            while len(dictionary) > k:
                dictionary[s[left]] -= 1
                if dictionary[s[left]] == 0:
                    del dictionary[s[left]] 
                left += 1 

            maxLen = max(maxLen, right - left + 1) 
        return maxLen
````

---

# Key Idea

This is a **variable-size sliding window** problem.

We want the **longest contiguous window** such that:

```python
number of distinct characters <= k
```

So we use two pointers:

* `left` = start of the current window
* `right` = end of the current window

We also use a hashmap `dictionary` to store the frequency of characters inside the current window.

---

# Why Sliding Window Works

As we move `right` forward, we expand the window by adding one character.

If the window becomes invalid, meaning it now contains **more than `k` distinct characters**, we shrink it from the left until it becomes valid again.

At every valid step, we compute the current window length and update the answer.

---

# What the Dictionary Stores

```python
dictionary[char] = number of times char appears in the current window
```

Example:

If current window is:

```python
"ece"
```

then dictionary is:

```python
{'e': 2, 'c': 1}
```

This tells us:

* the window contains 2 distinct characters
* frequencies are tracked so we know when a character fully leaves the window

---

# Step-by-Step Logic

## 1. Expand the window

For each `right`, add `s[right]` to the current window:

```python
dictionary[s[right]] += 1
```

---

## 2. Check if window is invalid

If the number of distinct characters becomes greater than `k`, the window is invalid:

```python
while len(dictionary) > k:
```

---

## 3. Shrink from the left

To make the window valid again:

* reduce the count of `s[left]`
* if its count becomes 0, remove it from the dictionary
* move `left` forward

```python
dictionary[s[left]] -= 1
if dictionary[s[left]] == 0:
    del dictionary[s[left]]
left += 1
```

---

## 4. Update answer

Now the window is valid again, so update the maximum length:

```python
maxLen = max(maxLen, right - left + 1)
```

---

# Why `right - left + 1`?

If the window spans indices from `left` to `right`, then its length is:

```python
right - left + 1
```

Example:

```python
left = 2
right = 5
```

Window covers indices:

```python
2, 3, 4, 5
```

That is 4 characters.

And:

```python
5 - 2 + 1 = 4
```

---

# Why We Remove Characters from the Dictionary

When a character's count becomes 0, it is no longer in the current window.

So we remove it:

```python
if dictionary[s[left]] == 0:
    del dictionary[s[left]]
```

This is important because:

```python
len(dictionary)
```

is how we count the number of distinct characters.

If we kept characters with count 0, then `len(dictionary)` would be wrong.

---

# Why `while` and Not `if`?

We use:

```python
while len(dictionary) > k:
```

because removing one character from the left may not be enough.

Example:

```python
s = "aaabbcc"
k = 2
```

If the window contains 3 distinct characters, we may need to shrink multiple times before it becomes valid again.

So `while` is necessary.

---

# Time Complexity

## O(n)

Each character is:

* added to the window once by `right`
* removed from the window at most once by `left`

So even though there is a nested `while`, the total work is still linear.

---

# Space Complexity

## O(k) to O(min(n, charset))

The dictionary stores the characters currently in the window.

In the worst case, it can store up to:

* `k + 1` characters briefly during shrinking
* or at most the size of the character set

So the usual description is:

```python
O(min(n, charset))
```

If you want to describe it simply, you can also say:

```python
O(k)
```

for the valid window state.

---

# Intuition in One Sentence

Expand the window to the right, and whenever it has more than `k` distinct characters, shrink it from the left until it becomes valid again.

---

# Pattern

This is the classic pattern:

> **Longest substring / subarray with at most K distinct elements**

Template:

```python
for right in range(len(arr)):
    add arr[right]

    while window is invalid:
        remove arr[left]
        left += 1

    update answer
```
