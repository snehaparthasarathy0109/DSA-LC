## LeetCode 881: Boats to Save People

You are given:

```python
people = [3, 2, 2, 1]
limit = 3
```

Each boat can carry:

```python
at most 2 people
```

and their total weight must be:

```python
<= limit
```

Return the minimum number of boats needed.

---

## Key idea

This is a **greedy two-pointer** problem.

Sort the people first:

```python
[1, 2, 2, 3]
```

Use two pointers:

```python
left = 0                 # lightest person
right = len(people) - 1  # heaviest person
```

For every boat, the **heaviest remaining person** must get on a boat.

Then ask:

```python
Can the lightest remaining person fit with the heaviest remaining person?
```

If yes, put them together.

If no, the heaviest person goes alone.

---

## Code

```python
from typing import List

class Solution:
    def numRescueBoats(self, people: List[int], limit: int) -> int:
        people.sort()

        left = 0
        right = len(people) - 1
        boats = 0

        while left <= right:
            # If the lightest and heaviest can fit together,
            # put both in the same boat.
            if people[left] + people[right] <= limit:
                left += 1

            # The heaviest person always gets placed in a boat.
            right -= 1

            # Count this boat.
            boats += 1

        return boats
```

---

## Why this works

After sorting:

```python
people = [1, 2, 2, 3]
```

Start with:

```python
left = 0   # weight 1
right = 3  # weight 3
```

Check:

```python
people[left] + people[right] = 1 + 3 = 4
```

`4 > 3`, so they cannot fit together.

That means `3` cannot pair with anyone else, because `1` is already the lightest person.

So `3` must go alone.

Move:

```python
right -= 1
boats += 1
```

Now:

```python
left = 0   # weight 1
right = 2  # weight 2
```

Check:

```python
1 + 2 = 3
```

They fit together, so put both in one boat.

Move:

```python
left += 1
right -= 1
boats += 1
```

Now only one `2` is left, so it gets its own boat.

Answer:

```python
3
```

---

## Why it is `if`, not `while`

Inside the loop, this should be an `if`:

```python
if people[left] + people[right] <= limit:
    left += 1
```

because each boat can carry **at most 2 people**.

Each iteration of the outer `while` loop represents exactly **one boat**.

So for one boat:

```python
heaviest person always goes
lightest person may also go
```

That is why we only move `left` once.

---

## Complexity

```python
Time: O(n log n)
Space: O(1)
```

Sorting takes `O(n log n)`.

The two-pointer scan takes `O(n)`.

So total time is:

```python
O(n log n)
```


We use:

```python
while left <= right:
```

because when `left == right`, there is **one person left**, and that person still needs a boat.

## Example

```python
people = [1, 2, 2]
limit = 3
```

Sorted:

```python
[1, 2, 2]
```

Start:

```python
left = 0   # 1
right = 2  # 2
boats = 0
```

`1 + 2 <= 3`, so they share one boat.

Now:

```python
left = 1
right = 1
boats = 1
```

There is still one person left:

```python
people[1] = 2
```

So we need one more boat.

That is why the loop must still run when `left == right`.

After that iteration:

```python
right -= 1
boats += 1
```

Now:

```python
left = 1
right = 0
boats = 2
```

The loop stops.

## Why not `left < right`?

If you used:

```python
while left < right:
```

then the loop would stop when one person is left, and you would forget to count their boat.

So:

```python
left < right
```

means “while there are at least two people left.”

But we need:

```python
left <= right
```

because we must handle both:

```python
two people left
one person left
```

Final rule:

```python
while left <= right:
```

means:

> Keep assigning boats while there is at least one person remaining.
