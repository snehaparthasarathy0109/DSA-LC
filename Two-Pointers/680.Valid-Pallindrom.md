## LeetCode 680: Valid Palindrome II

You need to check:

> Can `s` become a palindrome after deleting **at most one character**?

Key idea:

Use two pointers.
When characters match, keep moving inward.
When characters do **not** match, you get **one chance** to delete either:

1. the left character, or
2. the right character

Then check if the remaining substring is a palindrome.

---

## Solution

```python
class Solution:
    def validPalindrome(self, s: str) -> bool:
        def is_palindrome(left, right):
            while left < right:
                if s[left] != s[right]:
                    return False
                left += 1
                right -= 1
            return True

        left = 0
        right = len(s) - 1

        while left < right:
            if s[left] != s[right]:
                # Try deleting either s[left] or s[right]
                return (
                    is_palindrome(left + 1, right) or
                    is_palindrome(left, right - 1)
                )

            left += 1
            right -= 1

        return True
```

---

## Why this works

For a palindrome, characters at opposite ends must match.

Example:

```text
s = "abca"
```

Start:

```text
a b c a
^     ^
L     R
```

`a == a`, so move inward.

```text
a b c a
  ^ ^
  L R
```

Now:

```text
b != c
```

We are allowed to delete **one** character.

Try deleting `b`:

```text
a c a
```

This is a palindrome.

So return `True`.

---

## Dry run: `"abc"`

```text
a b c
^   ^
L   R
```

`a != c`

Try deleting left character `a`:

```text
b c
```

Not palindrome.

Try deleting right character `c`:

```text
a b
```

Not palindrome.

So return `False`.

---

## Important mistake to avoid

Do **not** just count mismatches.

This is wrong because deleting one character shifts the remaining characters.

Example:

```text
"abca"
```

There is one mismatch: `b != c`.

But after deleting `b`, the string becomes:

```text
"aca"
```

which is valid.

So we need to actually check both deletion possibilities.

---

## Complexity

```text
Time: O(n)
Space: O(1)
```

Even though we call `is_palindrome`, it only happens once at the first mismatch, so total work is still linear.
