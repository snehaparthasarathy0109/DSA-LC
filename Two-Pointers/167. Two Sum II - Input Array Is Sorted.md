# 167. Two Sum II - Input Array Is Sorted

## Problem
You are given a **1-indexed** array of integers `numbers` that is sorted in **non-decreasing order**, and an integer `target`.

You need to find **two numbers** such that they add up to `target` and return their **1-based indices** as:

```python
[index1, index2]
````

where:

* `1 <= index1 < index2 <= len(numbers)`

The problem guarantees that there is **exactly one solution**.

---

## Solution Code

```python
class Solution:
    def twoSum(self, numbers: List[int], target: int) -> List[int]:
        left = 0
        right = len(numbers) - 1

        while left < right:
            if numbers[left] + numbers[right] == target:
                return [left + 1, right + 1] 
            elif numbers[left] + numbers[right] < target:
                left += 1
            else:
                right -= 1
        return [-1, -1]
```

---

# Key Idea

This problem is solved using the **two pointers** pattern.

Because the array is already **sorted**, we can use two pointers:

* `left` starting at the beginning
* `right` starting at the end

At each step, we look at:

```python
numbers[left] + numbers[right]
```

and compare it with `target`.

---

# Why Sorting Helps

Since the array is sorted:

* moving `left` to the right makes the sum **larger**
* moving `right` to the left makes the sum **smaller**

That means we can adjust the sum efficiently without checking every pair.

This is much better than brute force.

---

# Step-by-Step Logic

## 1. Start with the two ends

```python
left = 0
right = len(numbers) - 1
```

So:

* `left` points to the smallest number
* `right` points to the largest number

---

## 2. Compute the sum

At each step:

```python
numbers[left] + numbers[right]
```

There are 3 possible cases.

---

## Case 1: The sum equals the target

```python
if numbers[left] + numbers[right] == target:
    return [left + 1, right + 1]
```

We found the answer.

We return `left + 1` and `right + 1` because the problem wants **1-based indices**, not 0-based indices.

---

## Case 2: The sum is too small

```python
elif numbers[left] + numbers[right] < target:
    left += 1
```

If the sum is too small, we need a larger number.

Since the array is sorted, moving `left` rightward gives us a bigger number.

So we increase `left`.

---

## Case 3: The sum is too large

```python
else:
    right -= 1
```

If the sum is too large, we need a smaller number.

Since the array is sorted, moving `right` leftward gives us a smaller number.

So we decrease `right`.

---

# Why This Works

Because the array is sorted, we can safely eliminate impossible pairs.

For example:

* if the current sum is too small, then using the same `left` with any index smaller than `right` will not help enough
* if the current sum is too large, then using the same `right` with any index larger than `left` will still be too large

So each move removes bad possibilities without missing the correct answer.

# Why Return `left + 1` and `right + 1`?

Python arrays use **0-based indexing**:

* first index is `0`
* second index is `1`

But this problem wants **1-based indexing**:

* first position is `1`
* second position is `2`

So we convert:

```python
return [left + 1, right + 1]
```

---

# Time Complexity

## O(n)

Each pointer only moves in one direction:

* `left` moves rightward
* `right` moves leftward

Each pointer moves at most `n` times total.

So the algorithm runs in:

```python
O(n)
```

---

# Space Complexity

## O(1)

We only use a few variables:

* `left`
* `right`

So the extra space is constant:

```python
O(1)
```

---

# Why This Is Better Than Brute Force

A brute force solution would check every pair:

```python
for i in range(n):
    for j in range(i + 1, n):
        ...
```

That takes:

```python
O(n^2)
```

But the sorted property allows us to do it in:

```python
O(n)
```

using two pointers.

---

# Pattern to Remember

This is the classic **opposite-direction two pointers** pattern.

You should think of this pattern when:

* the array is sorted
* you need a pair with some condition
* moving one pointer can predictably increase or decrease the answer

General template:

```python
left = 0
right = len(arr) - 1

while left < right:
    current = arr[left] + arr[right]

    if current == target:
        return answer
    elif current < target:
        left += 1
    else:
        right -= 1
```

---

# Intuition in One Sentence

Because the array is sorted, if the sum is too small we move `left` rightward, and if the sum is too large we move `right` leftward.
